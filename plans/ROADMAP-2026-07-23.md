## Strategic Note: Script Convergence

The Rust Node is not just a Windows port. It is a migration target for every
platform's operational scripts. Python and shell automation on macOS, Windows,
and Linux can converge into governed Rust capabilities shared across all
three platforms.

**Before convergence:**

```
macOS: Swift Core + Python scripts + shell scripts + automation glue
Windows: Rust Node + PowerShell scripts + NSSM config
Linux: (would repeat patterns from both)
```

**After convergence:**

```
Shared Rust capability layer (governed, tested, evidence-producing)
    ├── macOS adapter (launchd) — replaces Python ops scripts
    ├── Windows adapter (NSSM) — replaces PowerShell ops scripts
    └── Linux adapter (systemd) — built on same layer
```

Each promoted capability gains:
- Claim custody before execution
- Track ResidencyState during operation
- Collect evidence automatically
- Emit receipts on completion
- Full governance integration

This is the same pattern as MQR: existing operational logic becomes a governed
consumer rather than an ungoverned script.
